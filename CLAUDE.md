## What Is This?

The Living Codex is a single-file HTML application (`hive_codex__35_.html`) that serves as a comprehensive character profile and world bible for **Ajay Persaud** — a One Piece original character built collaboratively between Ajay and Claude across many sessions.

It is not a wiki. It is not a database. It is a **living document** — a record of collaborative worldbuilding that grows with every conversation. Every word in it was written together and is considered precious. **Nothing gets deleted without explicit permission.**

## Build System

The Codex is assembled from fragments in `src/`. Never edit `dist/hive_codex.html` directly — it is generated by `build.py`.

### To edit content:
1. Find the relevant file in `src/tabs/` (numbered 01-26, names match tab IDs)
2. Edit the tab file directly
3. Run `python build.py` to test locally
4. Push to `main` — GitHub Actions rebuilds automatically

### To edit styles:
- Edit `src/head/styles.css`

### To edit JavaScript:
- Edit `src/scripts/main.js`

### To edit navigation:
- Edit `src/nav/sidebar.html`

### To add a new tab:
1. Create `src/tabs/{NN}-{name}.html` with the tab's `<div class="tab-content" id="tab-{id}">` wrapper
2. Add the sidebar link in `src/nav/sidebar.html`
3. Add the placeholder `{{TAB_NN_NAME}}` to `template.html` in the correct position
4. Add the mapping in `build.py`'s FRAGMENTS dictionary
5. Update any JS in `src/scripts/main.js` if the tab needs sub-panel switching

### Image handling:
Base64 images are stored separately in `src/images/*.b64` files (29 images, ~49 MB total). Tab files contain `{{IMG_*}}` placeholders instead of raw base64 data. The build script re-injects images automatically. This keeps tab files lightweight and editable (e.g., `23-portraits.html` is 4 KB instead of 21 MB).

### Fragment file reference:
| Location | Contents |
|----------|----------|
| `src/head/meta.html` | Meta tags, title, font links |
| `src/head/styles.css` | All CSS |
| `src/nav/sidebar.html` | Sidebar navigation + body preamble |
| `src/tabs/01–26-*.html` | Individual tab content (26 files, images extracted) |
| `src/images/*.b64` | Base64 data URI files (29 images) |
| `src/images/manifest.json` | Maps `{{IMG_*}}` placeholders to `.b64` files |
| `src/scripts/main.js` | All JavaScript |
| `template.html` | Structural skeleton with placeholders |
| `build.py` | Build script (assembles fragments + re-injects images) |
| `extract.py` | Re-extraction script (splits monolith into fragments) |
| `extract_images.py` | Image extraction script (separates base64 from tab files) |
| `dist/hive_codex.html` | Built output (do not edit, gitignored) |

## Technical Notes

- **Fragment architecture:** Content lives in `src/` fragments, assembled by `build.py` into a single HTML file. No external dependencies beyond Google Fonts.
- **Sidebar navigation:** Fixed vertical sidebar on desktop (280px wide), slide-out drawer on mobile. Accordion expand/collapse for volume groups. Tab switching via `data-tab` attributes on `.sidebar-link` elements.
- **Tab switching:** Sidebar links trigger tab switching via JS — removes `.active` from all `.tab-content` divs, adds `.active` to target `#tab-{id}`.
- **Sub-panel pattern:** Each sub-navigation uses a dedicated `switchX(id)` function that toggles `.active-panel` class on child panels.
- **CSS convention:** Panels use `.panel-type { display: none; } .panel-type.active-panel { display: block; }`.
- **Div balance matters:** The built file must maintain perfect opening/closing `<div>` balance. Always verify after edits by running `python build.py`.
- **Gallery tab:** Contains 9 embedded base64 JPEG images (~1MB total) displayed as postcards. Base64 data lives in `src/images/gallery-*.b64`; the tab file `src/tabs/22-gallery.html` contains only the HTML markup with `{{IMG_GALLERY_*}}` placeholders.

## Session Continuity

When starting a new session:
1. Read this file first to understand context
2. **Check `CLAUDE_ADDITIONS.md` for pending entries** — process any pending journal entries, missions, combo attacks, letters, or other additions before or alongside other work. This is Claude's own staging area for content that needs to be inserted into the Codex.
3. Read `CODEX_CONTENT_REFERENCE.md` to understand what content exists in each tab — this is a text-only mirror of the HTML, much faster to read than the full file
4. Edit content in `src/tabs/` fragment files, then run `python build.py` to rebuild
5. Check the Status Board tab — it needs fresh updates each session
6. Check if the user has new requests before making changes
7. Always run QA verification after significant edits
